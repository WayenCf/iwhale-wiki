# 离线上传文件不扫描

> 来源：微服务平台团队-陈飞个人文档

---

## 问题现象

离线上传的文件未被扫描处理，对比正常和异常的 fileId：

| 状态 | fileId（Base64） | 解码后路径 |
|------|-----------------|-----------|
| ✔ 正常 | `L21udC9kbXMv...` | `/mnt/dms/crm/file/9fb8e649a62a/20251217/8/27/9-4670.xlsx` |
| ❌ 异常 | `L21udC9kbXMv...` | `/mnt/dms/crm/file//20260202/10/18/3-873.xlsx` |

**关键差异**：异常路径中 `file/` 后缺少机器编码（如 `9fb8e649a62a`），导致路径为 `file//` 双斜杠。

## 问题原因

云盘文件路径中 `CLOUD_NODE_ID`（机器编码）为空，导致路径拼接时出现 `file//`，文件无法被正确定位和扫描。

排查方式：
```bash
# 查找 Java 进程
ps -ef | grep java

# 检查 CLOUD_NODE_ID 环境变量
cat /proc/69/environ | tr '\0' '\n' | grep CLOUD_NODE_ID
```

## 解决方案

确保容器环境变量 `CLOUD_NODE_ID` 正确配置，使文件路径包含机器编码。

## 附：大账单 v2/make 接口超时

同一现场还遇到大账单 `v2/make` 接口调用超时问题：
- **原因**：应用管理配置的镜像为老镜像
- **方案**：更新为最新镜像后解决
