# 业务侧调用DMS生成PDF返回400 Bad Request

## 问题现象

业务侧（`irc-web-product`，计费系统）通过 `SimulationEntryController.createConsolidatebillPdf` 调用 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 生成合并账单 PDF，返回 400 Bad Request。

### 关键日志

```
ERROR|PortalUrlAccessControl.java:101|check|user not login in
WARN|ResourceBundleMessageSource.java:218|ResourceBundle [i18n/base] not found for MessageSource
DEBUG|SimulationEntryController.java:1305|createConsolidatebillPdf|billId is 31125016
DEBUG|DmsRestFulUtil.java:813|------URL is http://10.10.20.47:443/portal-web/dms/doc/L21udC9kbXMv...?regenerate=true
ERROR|SimulationEntryController.java:1333|createConsolidatebillPdf failed, invoice file read and write error.
org.springframework.web.client.HttpClientErrorException$BadRequest: 400 Bad Request on GET request for "http://10.10.20.47:443/portal-web/dms/doc/L21udC9kbXMv...": [no body]
```

## 问题原因

**HTTP/HTTPS 协议不匹配**：`DmsRestFulUtil` 构造的请求 URL 使用了 `http://`，但目标端口 443 是 HTTPS 端口，服务端只接受 HTTPS 请求。

- 请求 URL：`http://10.10.20.47:443/portal-web/dms/doc/...`（❌ HTTP 协议访问 HTTPS 端口）
- 正确 URL：`https://10.10.20.47:443/portal-web/dms/doc/...`（✅ HTTPS 协议）

用 HTTP 协议请求 HTTPS 端口，服务端无法解析 HTTP 格式的请求，直接返回 400 Bad Request（无响应体）。

### 调用链路

```
irc-web-product（计费系统）
  → SimulationEntryController.createConsolidatebillPdf()
    → DmsRestFulUtil.getByteByInvoicingId()
      → 构造 URL: http://10.10.20.47:443/portal-web/dms/doc/{base64(invoiceId)}?regenerate=true
        → RestTemplate.execute()
          → 400 Bad Request [no body]
```

### URL 构造细节

| 参数 | 值 |
|------|-----|
| 协议 | `http://`（❌ 应为 `https://`） |
| 主机 | `10.10.20.47` |
| 端口 | `443`（HTTPS 标准端口） |
| 路径 | `/portal-web/dms/doc/{base64}` |
| 认证 | HTTP Basic Auth（用户 `billing`，密码 MD5 哈希） |
| 参数 | `regenerate=true` |

## 解决方案

将 `DmsRestFulUtil` 中构造 URL 的协议从 `http://` 改为 `https://`，或检查相关配置项确保协议与端口一致。

## 关键要点

- **端口 443 是 HTTPS 标准端口**，用 HTTP 协议请求 HTTPS 端口必然返回 400
- `DmsRestFulUtil` 构造 URL 时，协议应与目标服务端口匹配
- 400 Bad Request [no body] 是 HTTP/HTTPS 协议不匹配的典型表现
- 类似问题在 [[wiki/dms90/来源/2025-09-23 阿富汗ET现场容灾切换]] 中也出现过：DMS 切换 HTTPS 后，调用方未同步修改协议导致请求失败
