# 马来TM数据库主键冲突问题

## 问题现象

马来 TM 项目现场 SIT 环境，创建客户的时候上传文件报错，日志中打印主键冲突：

```
com.iwhalecloud.ftf.dms.repo.api.exception.DuplicatePKException: Duplicate entry for key field: [File.id]
```

提示信息：`This is most probably caused by the lagging of database sequence`

## 问题分析

初步分析原因是数据库导入数据导致序列缓存未更新。

## 序列检查 SQL

```sql
SELECT * FROM USER_SEQUENCES; -- 序列检查
SELECT DMS_FILE_ID_SEQ.NEXTVAL SEQ FROM DUAL; -- 序列+1
SELECT ID FROM DMS_FILE WHERE REGEXP_LIKE(id, '^\\d+$') ORDER BY TO_NUMBER(ID) DESC; -- 获取最大序列数值
```

## 问题解决

现场人员重启 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 应用后解决。

## 关键要点

- 数据导入后 序列 缓存可能未同步更新
- [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 内部缓存了 序列 值
- 重启应用可刷新 序列 缓存
