# DMS菜单权限优化

## 需求背景

1. [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 的 Home 页面原本 Admin 进去可以看到 Configuration 页面，用于配置账号权限，目前 Admin 进去也看不到了
2. 现场需要从生产环境导出某目录下所有模版，需要一个账号有所有模版的查看/下载权限

## 修改方案

### 后端修改逻辑

1. 根据用户角色返回需要展示的菜单列表：
   - `user_role == ROLE_ADMIN`：返回 `["Home","Template","Label","Configuration","Certificate","Inquiry&Statistic"]`
   - `user_role != ROLE_ADMIN`：返回 `["Home","Template","Label","Certificate","Inquiry&Statistic"]`

2. 修改当前接口逻辑：
   - 保留根据服务类型判断是否显示 Document 标签的逻辑
   - 删除根据部署模式判断是否显示 Configuration 标签逻辑

3. 修复接口返回数据格式（原为字符串不是数组格式导致前台解析报错）

### 前端修改逻辑

删除根据用户角色是否为 `ROLE_ADMIN` 进行二次回调的逻辑，前台根据 `/menu/list` 接口返回数据展示标签

## 验证结果

- Admin 账号登录可以访问 Configuration 界面，且可以配置其他用户的权限
- 非 Admin 用户不显示 Configuration 菜单
