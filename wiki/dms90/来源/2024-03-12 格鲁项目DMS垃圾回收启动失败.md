# 格鲁项目DMS垃圾回收启动失败

## 问题现象

JVM 启动失败，报错提示无法创建 Java 虚拟机，涉及垃圾收集器配置冲突。

## 问题原因

- 基础镜像中的 JDK 默认带有一个 ConcMarkSweepGC 垃圾收集器
- 环境变量配置了 [[wiki/dms90/概念/G1GC垃圾回收器]] 垃圾收集器
- JVM 启动不允许同时配置两个垃圾收集器，导致启动失败

## 解决方案

1. 从环境变量 `JAVA_OPTS` 中删除 `-XX:+UseG1GC`
2. 新增环境变量 `COVER_OPTS=-XX:+UseG1GC` 强制指定使用 G1GC

## 配置示例

```bash
JAVA_OPTS
-Xms8192m -Xmx8192m -Xmn4096m -XX:MetaspaceSize=512m -XX:LargePageSizeInBytes=128m -XX:+HeapDumpOnOutOfMemoryError -DZSMART_HOME=/app/ZSMART_HOME -Djavax.xml.parsers.DocumentBuilderFactory=com.sun.org.apache.xerces.internal.jaxp.DocumentBuilderFactoryImpl -XX:+PrintGC -XX:+PrintGCDetails -XX:+PrintGCTimeStamps -XX:+PrintGCDateStamps -XX:+PrintHeapAtGC -Xloggc:/app/log/gc.log

COVER_OPTS
-XX:+UseG1GC
```

## 关键要点

- 使用 `COVER_OPTS` 环境变量可以覆盖 JVM 默认配置
- 不同 JDK 版本默认垃圾收集器可能不同
- 需要确保垃圾收集器配置的唯一性
