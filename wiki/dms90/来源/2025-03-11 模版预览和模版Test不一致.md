# 模版预览和模版Test不一致

## 问题背景

使用模版 runtime 操作时：
- 预览模版，字体是加粗的
- Test 模版，字体没有加粗

## 问题分析

分析 test 和预览的接口调用：
- 预览接口调用 `release/getSnapshot`，获取当前最新模版
- Test 调用的是生成模版接口 `make`，但没有获取到最新模版

## 问题原因

历史账单需求设计中，前端传入了当前模版时间，导致 Test 接口没有获取到最新模版。

## 解决方案

在 make 接口断点处新增历史模块开关判断，当关闭时忽略入参时间：

```
提单解决：#11377737 DMS模版预览和模版test优化
```

修改代码后，点击 Test 模版可以获取到最新模版。

## 关键要点

- 预览获取最新模版，Test 可能使用历史模版
- 历史账单功能会传入时间参数影响模版选择
- 需要增加开关控制是否使用历史模版
