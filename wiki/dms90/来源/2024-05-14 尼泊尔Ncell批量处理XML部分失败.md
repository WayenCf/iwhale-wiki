# 尼泊尔Ncell批量处理XML部分失败

## 问题现象

批量出账 XML 处理中部分账单报错失败。

## 问题分析

业务生成的 XML 中包含 `&` 特殊字符，解析 XML 报错：

```
org.xml.sax.SAXParseException: The entity name must immediately follow the '&' in the entity reference.
```

XML 规范中 `&` 是特殊字符，必须用 `&amp;` 转义。

## 解决方案

1. 修改业务侧生成 XML 代码，将 `&` 字符转义为 `&amp;`
2. 或在 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 批量处理逻辑中增加 XML 特殊字符的预处理

## 关键要点

- XML 特殊字符（`&`, `<`, `>`, `"`, `'`）需要正确转义
- `&` 必须转义为 `&amp;`
- 批量出账时需注意业务侧数据中的特殊字符
