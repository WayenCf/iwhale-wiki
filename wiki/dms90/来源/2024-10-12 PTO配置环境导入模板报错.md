# PTO配置环境导入模板报错

## 问题现象

导入模版报错，使用的数据库是 [[wiki/dms90/实体/TeleDB]]。

## 问题分析

本地使用环境的数据库配置能复现问题。日志报错显示服务端因为未知原因主动关闭了连接。

报错关键信息：

```
The last packet successfully received from the server was 681 milliseconds ago.
The last packet sent successfully to the server was 520 milliseconds ago.
```

## 解决方案

1. 在 [[wiki/dms90/实体/TeleDB]] 管理平台查到数据库集群
2. 将配置文件中的连接信息修改为集群中另一个节点，问题解决
3. 联系集成协助排查，集成让重启数据库，重启后使用原先的节点功能也正常了

## 关键要点

- [[wiki/dms90/实体/TeleDB]] 集群部署时需注意连接节点可用性
- 数据库连接中断可能由节点故障导致
- 切换到集群中其他节点可临时解决
