# 新电生产环境发布模版报错序列问题

## 问题现象

生产环境发布模版报错，序列值与实际数据不一致。

## 问题分析

```sql
SELECT * FROM tfm_sequences;
SELECT * FROM dms_release ORDER BY creation_date DESC;
```

- `release` 表最大 ID 序列记录为 22001
- `tfm_sequences` 表的当前序列值为 10000

## 问题原因

1. 生产环境签过模版数据（`release` 表数据），ID 为 22001 的数据是迁移进来的
2. 迁移未做 `tfm_sequences` 表的更新
3. 发布模版时取序列表记录自增，发现存在相同序列 ID，导致报错

## 解决办法

手动调整序列信息：

1. 执行 `SELECT * FROM dms_release ORDER BY creation_date DESC`，查询出最大序列 ID
2. 执行 `UPDATE tfm_sequences SET current_value=xxx WHERE sequence_name='DMS_RELEASE_ID_SEQ'`，xxx 为最大 ID + 10000

## 最终建议

项目在迁移模版数据时，同步迁移 `tfm_sequences` 表相关 DMS 开头的 sequence 信息。

## 关键要点

- 数据迁移时必须同步更新 序列 表
- `tfm_sequences` 表存储 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 各表的 ID 序列
- 序列值必须大于已有数据的最大 ID
