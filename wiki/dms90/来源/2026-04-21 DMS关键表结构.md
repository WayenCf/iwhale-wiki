# DMS 关键表结构（Oracle 关系模型）

> DMS 在 Oracle/TeleDB 中维护的 5 张核心关系表，支撑模版管理、发布流程、租户隔离和用户认证。

## 表总览

| 表名 | 主键 | 核心职责 | 关联关系 |
|------|------|---------|---------|
| `DMS_USER` | ID (VARCHAR2) | 用户账号与角色 | → `DMS_TENANT`.ID (外键) |
| `DMS_TENANT` | ID (VARCHAR2) | 租户与存储配额 | ← `DMS_USER`.TENANT |
| `DMS_RESOURCE` | ID (VARCHAR) | 模版资源元数据 | → `dms_resource_attach` (FILE_ID 外键) |
| `DMS_RELEASE` | ID (VARCHAR2) | 发布记录 | ← `DMS_USER`.ID (USER_ID) |
| `DMS_RELEASE_RESOURCE` | (RELEASE_ID, ID) 联合主键 | 发布资源明细 | → `DMS_RELEASE`.ID (RELEASE_ID 外键) |

## DMS_USER — 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| ID | VARCHAR2 | **用户名**（主键） |
| PASSWORD | VARCHAR2 | 密码 |
| UPDATE_DATE | TIMESTAMP | 更新时间 |
| CREATION_DATE | TIMESTAMP | 创建时间 |
| USER_ROLE | VARCHAR2 | 角色信息 |
| EMAIL | VARCHAR2 | 邮箱 |
| ENABLED | CHAR | 是否启用 |
| TENANT | VARCHAR2 | 所属租户（**外键 → DMS_TENANT.ID**） |
| CRT | VARCHAR2 | 最后修改记录的用户 ID |

> `DMS_USER.ID` 对应 `DMS_RELEASE.USER_ID`，即用户发布的记录通过此关联追溯。

## DMS_TENANT — 租户表

| 字段 | 类型 | 说明 |
|------|------|------|
| ID | VARCHAR2 | 租户 ID（主键） |
| UPDATE_DATE | TIMESTAMP | 最近更新时间 |
| TOTAL_SIZE | NUMBER | 存储空间总大小 |
| CREATION_DATE | TIMESTAMP | 创建时间 |
| USED_SIZE | NUMBER | 已使用存储空间大小 |

> 一个租户下包含多个用户（`DMS_USER.TENANT` → `DMS_TENANT.ID`），实现存储配额的租户级管理。

## DMS_RESOURCE — 模版资源表

| 字段 | 类型 | 说明 |
|------|------|------|
| ID | VARCHAR | **前台模版路径**（主键） |
| UPDATE_DATE | TIMESTAMP | 更新日期 |
| DESCRIPTION | VARCHAR2 | 描述（允许 null） |
| LABEL | VARCHAR2 | 标签 |
| CREATION_DATE | TIMESTAMP | 创建时间 |
| USER_ID | VARCHAR2 | 用户 |
| VERSION | NUMBER | 版本 |
| DEPENDENCIES | CLOB | 依赖关系（允许 null） |
| PERMISSION_MASK | NUMBER | 权限掩码 |
| RESOURCE_TYPE | VARCHAR2 | 资源类型 |
| FILE_ID | VARCHAR2 | 文件 ID（外键 → `dms_resource_attach`） |
| RELEASE_VERSION | VARCHAR2 | 发布版本（允许 null） |
| CODE | VARCHAR2 | 代码信息（允许 null） |

> **ID 即模版路径**：这是 DMS 的关键设计——模版在前台的访问路径就是数据库主键。
> **DEPENDENCIES 字段**：CLOB 类型，存储依赖项 ID，超长时会触发 ORA-22275 错误（见 [[wiki/dms90/概念/模版导入]]）。

## DMS_RELEASE — 发布记录表

| 字段 | 类型 | 说明 |
|------|------|------|
| ID | VARCHAR2 | 主键 |
| CREATION_DATE | TIMESTAMP | 创建时间 |
| UPDATE_DATE | TIMESTAMP | 更新时间 |
| USER_ID | VARCHAR2 | 发布用户 ID（→ `DMS_USER.ID`） |
| DESCRIPTION | VARCHAR2 | 发布描述 |

> `DMS_RELEASE.ID` 对应 `DMS_RELEASE_RESOURCE.RELEASE_ID`，实现发布→资源的级联关系。
> 查询最新发布：先从 `DMS_USER` 查出 `#{tenant}` 对应用户 ID，再从 `DMS_RELEASE` 按创建时间降序取第一行。

## DMS_RELEASE_RESOURCE — 发布资源明细表

| 字段 | 类型 | 说明 |
|------|------|------|
| RELEASE_ID | VARCHAR2 | 所属发布 ID（外键 → `DMS_RELEASE.ID`） |
| ID | VARCHAR2 | 模版路径（联合主键之一） |
| FILE_ID | VARCHAR2 | 文件 ID |
| LABEL | VARCHAR2 | 标签 |
| DESCRIPTION | VARCHAR2 | 描述 |
| USER_ID | VARCHAR2 | 发布用户 ID |
| UPDATE_DATE | TIMESTAMP | 更新时间 |
| CREATION_DATE | TIMESTAMP | 创建时间 |
| RELEASE_VERSION | VARCHAR2 | 发布版本号 |
| RESOURCE_TYPE | VARCHAR2 | 资源类型 |
| DEPENDENCIES | CLOB | 依赖项 ID |
| CODE | VARCHAR2 | 代码信息 |

> **联合主键**：(RELEASE_ID, ID)，一个发布可包含多个模版资源。

## 表间关联图

```
DMS_TENANT ←── DMS_USER ──→ DMS_RELEASE ──→ DMS_RELEASE_RESOURCE
    (1:N)          (1:N)           (1:N)

DMS_RESOURCE ──→ dms_resource_attach
    (FILE_ID 外键)
```

## 与 MongoDB 存储的分工

| 存储层 | 负责内容 | 表/集合 |
|--------|---------|---------|
| **Oracle/TeleDB** | 元数据、用户、租户、模版资源、发布记录 | 上述 5 表 + 日志表 |
| **MongoDB** | 文档内容、文件二进制、任务记录 | `Document-{shardKey}`、`File-{shardKey}`、GridFS |

> 详见 [[wiki/dms90/概念/双存储适配]] 和 [[wiki/dms90/实体/MongoDB]]。
