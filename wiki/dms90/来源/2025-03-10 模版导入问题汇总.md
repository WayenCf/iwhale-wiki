# 模版导入问题汇总

## 问题一：CSRF报错

### 问题现象

控制台打印 CSRF 报错。

### 问题原因

配置项未配置 CSRF 跳过规则：

```properties
app.security.csrf-url-skip=/doc.*|/release.*|/prerelease.*|/publish.*|/delegate.*|/doc.*|/file.*|/folder.*|/material.*|/search.*|/resources.*|/label.*|/metadata.*|/dms/services.*|/dms/rest_v2.*|/dms/j_spring_security_check.*|/dms/crt.*|/crt.*
```

DMS 标准配置未配置 skip，关闭开发者模式后需要添加。

## 问题二：SQL连接问题

### 问题现象

接口报 SQL 连接错误，报错信息与 [[wiki/dms90/实体/TeleDB]] 连接中断相关。

### 问题原因

[[wiki/dms90/实体/TeleDB]] 管理平台查到数据库集群中 DMS 配置的节点挂了，切换到另一个节点解决。

## 问题三：Oracle CLOB字段错误

### 问题现象

接口报 SQL 执行错误：

```
ORA-22275: 指定的 LOB 定位符无效
```

### 问题原因

更新表字段内容太大，[[wiki/dms90/实体/Oracle]] `CLOB` 字段报错。`DMS_RESOURCE` 表的 `DEPENDENCIES` 字段内容超长。

### 解决方案

暂无代码级解决办法，本地手动在数据库中更新处理。

## 关键要点

- [[wiki/dms90/概念/CSRF配置|CSRF]] 跳过规则需要在标准配置中添加
- [[wiki/dms90/实体/TeleDB]] 节点故障需切换连接
- [[wiki/dms90/实体/Oracle]] `CLOB` 字段超长需手动处理
