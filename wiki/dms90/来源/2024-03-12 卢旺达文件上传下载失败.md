# 卢旺达文件上传下载失败

## 文件上传问题

### 问题现象

文件上传到 [[wiki/dms90/实体/云盘]] 时报错，访问路径 `/mnt/dms/custc/file/` 不存在。

### 问题原因

- [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 的 Dockerfile 定义用户为 `zpaa`
- 安全改造时只给 `zpaas` 用户授权
- 业务应用的用户未统一（有两个 `root`、一个 `zpaas`）

### 解决方案

将所有应用用户统一改为 `zpaas`

### 故障排查建议

遇到文件上传问题时应检查：
1. [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 应用是否为 `zpaas` 用户
2. `zpaas` 用户是否有读写权限
3. [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 挂载的 [[wiki/dms90/实体/云盘]] 容量剩余大小，若满了需要扩容

## 文件下载问题

### 问题现象

现场业务侧提示文件找不到，批量生产模版接口卡住不执行。

### 问题分析

批量处理流程：
1. 业务侧生成模版 XML 文件
2. 上传到与 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 主机共享目录
3. [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 批量处理逻辑扫描共享目录
4. 移动共享目录 XML 到 workspace 待处理目录
5. 删除共享目录 XML
6. 代码记录 `DMS_TASK_RECORD` 表状态为 `running`
7. 代码获取 XML 文件并调用 make 生成模版文档接口
8. 修改记录 `DMS_TASK_RECORD` 表状态为 `success`

### 问题原因

- 模版 `CUST_BILL_ADDR` 字段内容超长
- 导致后台 `JRBaseFiller.fillf` 方法执行 waiting 死循环
- 只有 debug 日志可以看到，现场未查到报错

### 解决方案

1. 缩减 `CUST_BILL_ADDR` 字段内容，或模版设计处进行调整
2. 删除 `task_record` 表状态为 `running` 的记录
3. 重启 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 服务
4. 业务前台重新操作

### 故障排查思路

1. 让业务侧重新执行测试用例（文件可能被删除）
2. 检查 `task_record` 表是否有 `running` 状态
3. 获取现场提供的模版，本地验证 make 接口是否能生成模版内容
