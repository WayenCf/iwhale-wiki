# MPT模版设计器预览与接口生成效果不一致

## 问题现象

模版通过设计器预览和实际调用接口生成（或 Web 界面预览）效果不一致，存在文本被截断的情况。

## 问题分析

推测可能是 [[wiki/dms90/实体/JasperReports]] 后台计算的文字宽度与 PDF 里实际的宽度不一致，导致文本提前换行。超过 [[wiki/dms90/实体/JasperReports]] 计算出的文本所需行数的部分被截断。

## 解决方案

为模版配置属性：

```properties
net.sf.jasperreports.export.pdf.force.linebreak.policy=true
```

此配置强制 [[wiki/dms90/实体/JasperReports]] 使用 PDF 渲染时的实际行宽进行换行判断，避免文字截断。

## 关键要点

- [[wiki/dms90/实体/JasperReports]] 文字宽度计算与 PDF 实际渲染可能不一致
- `force.linebreak.policy=true` 强制使用 PDF 行宽
- 设计器预览与实际生成的差异通常与字体渲染相关
