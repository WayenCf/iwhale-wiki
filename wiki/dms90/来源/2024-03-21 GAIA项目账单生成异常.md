# GAIA项目账单生成异常

## 问题一：模版未上传导致出账失败

### 问题现象

业务应用调用 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 打印接口报错。

### 问题原因

- 数据库 resource 表中模版已存在
- 但 resource 表的用户不一致
- 测试环境使用 A 用户设计模版，导入其他环境后没有 A 用户
- 导致模版无法生成账单

### 临时解决办法

修改 resource 表的用户信息为 `admin`

### 最终解决办法

设计模版使用特殊账号，保证在各个环境都存在当前用户

## 问题二：账单下载后空白

### 问题现象

业务调用 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 的 make 接口，账单下载后是空白，[[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 没有报错日志。

### 问题原因

[[wiki/dms90/实体/JasperReports]] 渲染时，XML 和模版不匹配，导致渲染后的账单是空白。

### 解决方案

联系配置组，检查模版中的字段和 XML 中的节点是否匹配，若不匹配则更换 XML 或者更换与 XML 相匹配的模版。

## 关键要点

- [[wiki/dms90/概念/模版导入]] 时需注意用户一致性
- 空白账单通常是 XML 与模版字段不匹配导致
- [[wiki/dms90/实体/JasperReports]] 渲染失败不一定会在日志中报错
