# JDK17不保存方法参数名称引发报错

## 问题现象

请求接口报错，无 body 返回。

## 问题分析

分析代码发现 `@PathVariable` 写法不规范：
- JDK 8 中支持不指定参数名称
- JDK 17 之后默认不保存方法参数名称，导致报错

## 解决方案

全局搜索 `@PathVariable`，为所有注解添加参数名称：

```java
// 修改前
@PathVariable String id

// 修改后
@PathVariable("id") String id
```

## 关键要点

- JDK 8 升级到 JDK 17 需注意参数名称保留问题
- `@PathVariable`、`@RequestParam` 等注解需显式指定参数名
- 可通过编译参数 `-parameters` 保留方法参数名称
