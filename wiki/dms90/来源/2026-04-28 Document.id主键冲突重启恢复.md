# Document.id 主键冲突重启恢复

> DMS 生成文档时出现 `Duplicate entry for key field: [Document.id]`，根因是内部序列缓存与 MongoDB 数据不同步，重启应用即可恢复。

## 问题现象

调用 DMS 生成文档接口返回 500 错误：

```
POST https://10.10.178.184/portal-web/dms/doc/make

错误码：7062101
错误信息：Duplicate entry for key field: [Document.id]
异常类型：HttpServerErrorException$InternalServerError
```

## 根因分析

1. DMS 使用 `idGenerator.get()` 生成 ObjectId，内部缓存了当前序列值
2. 当 MongoDB 数据被外部操作（数据迁移、手动插入等）后，程序仍使用缓存的旧序列值
3. 生成的 ID 与已有记录冲突 → MongoDB error code 11000 → `DuplicatePKException`
4. 异常被全局处理器映射为 500 Internal Server Error

## 解决方案

**重启 DMS 应用**即可恢复。重启后 `idGenerator` 重新从 MongoDB 获取当前最大 ID，缓存自动刷新。

## 排查步骤

1. 确认错误日志中有 `Duplicate entry for key field: [Document.id]`
2. 查询 MongoDB 确认冲突 ID：`db.getCollection("Document-{shardKey}").find({_id: "冲突的ID"})`
3. 重启 DMS 应用
4. 根本排查：是否有数据迁移或手动插入操作导致序列不同步
