# 阿富汗ETA离线生成账单功能失败

## 问题背景

阿富汗 ETA 现场离线生成账单功能失败。

## 问题分析

离线生成问题不好排查，需在本地模拟复现：

1. 获取环境下的模版信息（`ETA_BILL.jrxml`），导入到本地环境并发布
2. 获取现场的 XML 配置，同步到本地（XML 配置规则现场定制）
3. 本地应用启动后，优先放入成功的 XML，确认正常跑完
4. 检查本地 `dms_task_record` 表状态从 `running` 到 `succeeded`
5. 检查 `dms_document` 表，账单都生成了
6. 重复步骤放入失败的 XML，观察数据库记录状态和控制台日志
7. 确认执行都是成功的
8. 计数对比：2个 XML 分别 1000 个账单，查询 Document 表生成了 2000 个账单

## 问题结论

确认模版和 XML 文档，离线功能正常。问题可能出在现场环境配置或数据上。

## 关键要点

- 离线账单生成可通过本地模拟验证
- 需要获取现场模版和 XML 配置
- 检查 `dms_task_record` 和 `dms_document` 表状态确认结果
- 本地验证正常则问题在环境配置
