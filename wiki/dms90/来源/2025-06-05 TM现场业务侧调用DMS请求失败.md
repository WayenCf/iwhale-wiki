# TM现场业务侧调用DMS请求失败

## 问题背景

现网有一笔第三方 payment 交易，调用 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 接口报错：

```
Invoke dms error, url:http://172.18.1.210:80/portal-web/dms/doc/make
```

## 问题分析

1. 确认业务侧代码调用 http 接口返回错误码 45366190
2. 跟踪业务网关报错情况：
   - 业务每秒调用 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 接口 2 次
   - 38 秒时没有请求
   - 检查 error 日志发现网关做了 nginx reload 动作

## 问题原因

nginx reload 导致业务侧请求 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 失败：
- nginx 配置类型为模版刷新
- 当有应用节点发生改变（如应用重新部署），nginx 执行 reload
- `mccm-evtservice-product@pr` 应用在 37 秒时重启导致 nginx reload

## 最终建议

TM 现场业务侧网关配置类型模版刷新，需要考虑 [[wiki/dms90/来源/2026-04-19 DMS项目概览|DMS]] 相同业务网关下的应用做重新部署和升级会导致请求失败。

## 关键要点

- nginx reload 会短暂中断请求
- 模版刷新类型网关在应用部署时自动 reload
- 相同网关下的应用部署需协调时间窗口
