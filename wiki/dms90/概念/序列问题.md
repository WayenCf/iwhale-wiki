# 序列问题

> DMS 使用数据库序列生成主键 ID，序列与实际数据不一致时会导致主键冲突。

## 在现场问题中的角色

### 序列缓存未更新

- [[wiki/dms90/来源/2024-04-08 马来TM数据库主键冲突问题|马来TM项目数据库主键冲突问题]]：数据库导入数据导致序列缓存未更新，重启应用解决

### DMS 内部缓存序列值

- [[wiki/dms90/来源/2025-09-26 DMS序列问题|DMS序列问题]]：DMS 内部缓存了序列值，修改数据库后程序取到的仍是缓存值，需重启刷新

### 数据迁移未同步序列

- [[wiki/dms90/来源/2025-06-10 新电生产环境发布模版序列问题|新电生产环境发布模版序列问题]]：迁移 release 表数据但未同步 `tfm_sequences` 表

### 发布序列与表关联

- `DMS_RELEASE` 表使用 `DMS_RELEASE_ID_SEQ` 序列生成主键
- `DMS_RELEASE_RESOURCE` 使用联合主键 (RELEASE_ID, ID)，不依赖独立序列
- 迁移发布数据时必须同步 `tfm_sequences` 中对应序列值
- 详见 [[wiki/dms90/来源/2026-04-21 DMS关键表结构|DMS 关键表结构]]

## 排查步骤

1. 检查当前序列值：`SELECT DMS_FILE_ID_SEQ.NEXTVAL FROM DUAL;`
2. 查询最大 ID：`SELECT ID FROM DMS_FILE ORDER BY TO_NUMBER(ID) DESC;`
3. 若序列值小于最大 ID，需手动调整

## 调整方法

### 方法一：逐步递增

```sql
-- 每次执行 NEXTVAL 让序列递增，直到超过最大 ID
SELECT DMS_FILE_ID_SEQ.NEXTVAL FROM DUAL;
```

### 方法二：调整步长

```sql
-- 临时增大步长快速跳过
ALTER SEQUENCE DMS_FILE_ID_SEQ INCREMENT BY 10000;
-- 执行几次 NEXTVAL
SELECT DMS_FILE_ID_SEQ.NEXTVAL FROM DUAL;
-- 调回原步长
ALTER SEQUENCE DMS_FILE_ID_SEQ INCREMENT BY 1000;
```

### 方法三：直接更新 tfm_sequences

```sql
UPDATE tfm_sequences SET current_value=xxx WHERE sequence_name='DMS_RELEASE_ID_SEQ';
-- xxx = 最大 ID + 10000
```

## 关键要点

- DMS 内部缓存序列值，修改数据库后必须重启应用
- 数据迁移时必须同步 `tfm_sequences` 表
- 序列值必须大于已有数据的最大 ID
- 重启应用是最简单的刷新方式
