# 异常体系

> DMS 的异常分为三大分支：Business（业务异常）、System（系统异常）、MetricBlock（流控阻断），配合全局异常处理器统一映射到 HTTP 状态码。

## 三大异常分支

```
DmsException（抽象基类）
  ├── DmsBusinessException（业务异常，400）
  ├── DmsSystemException（系统异常，500）
  └── MetricBlockException（流控阻断，500）
```

## DmsBusinessException（业务异常）

### 说明

- HTTP 状态码：400 BAD_REQUEST
- 用途：业务逻辑错误，如参数校验失败、权限不足、资源不存在

### 子类（dms-common/exception 包）

| 异常类 | 场景 |
|--------|------|
| FolderException | 文件夹操作异常（父目录不存在、同名冲突） |
| InvalidCertificateException | 证书无效（格式错误、过期） |
| MaterialNotFoundException | 文件/素材不存在 |
| UnAuthorizedException | 未授权访问（跨租户、无权限） |
| UnInitializedException | 系统未初始化（无 Release、无租户） |

## DmsSystemException（系统异常）

### 说明

- HTTP 状态码：500 INTERNAL_SERVER_ERROR
- 用途：系统级错误，如数据库连接失败、网络超时、配置错误

### 子类（dms-common/exception 包）

| 异常类 | 场景 |
|--------|------|
| DMSMongoException | MongoDB 操作失败（连接超时、查询超限） |
| RedisException | Redis 操作失败 |
| SpoolUploadException | 云盘上传失败 |
| MqException | MQ 消息发送失败 |
| DuplicatePKException | 主键冲突（MongoDB error code 11000） |

## MetricBlockException（流控阻断）

### 说明

- HTTP 状态码：500 INTERNAL_SERVER_ERROR
- 用途：Sentinel 流控阻断（并发超限、熔断降级）

### 触发场景

- 写操作连续异常超过阈值（默认 100 次）
- 读操作连续异常超过阈值（默认 200 次）
- 写操作平均响应时间超过阈值（默认 60000ms）
- 读操作平均响应时间超过阈值（默认 30000ms）

## dms-web 专用异常（高性能优化）

### 异常列表

| 异常类 | HTTP 状态码 | 特性 |
|--------|------------|------|
| ContentLengthMismatchException | 400 | fillInStackTrace() 跳过堆栈填充 |
| UploadCapacityExceededException | 429 | fillInStackTrace() 跳过堆栈填充 |
| UploadTooManyRequestsException | 429 | fillInStackTrace() 跳过堆栈填充 |
| ContentLengthRequiredException | 411 | fillInStackTrace() 跳过堆栈填充 |

### 优化目的

- 覆写 `fillInStackTrace()` 返回 `this`
- 高并发下节省每次约 5-20us 的 native 堆栈填充开销

## 全局异常处理器（AppExceptionHandler）

### @ControllerAdvice 映射表

| 异常类型 | HTTP 状态码 | 业务场景 |
|---------|------------|---------|
| Exception（兜底） | 500 | 未预期的服务端错误 |
| MaxUploadSizeExceededException | 413 | 上传文件超过大小阈值 |
| MetricBlockException | 500 | Sentinel 降级拦截 |
| DmsSystemException | 500 | 系统级异常 |
| DmsBusinessException | 400 | 业务异常 |
| MaterialNotFoundException | 404 | 素材文件未找到 |
| MissingServletRequestPartException | 400 | 请求缺少必要部分 |
| ContentLengthRequiredException | 411 | 缺少 Content-Length 头 |
| UploadTooManyRequestsException | 429 | 上传并发数超限 |
| UploadCapacityExceededException | 429 | 上传总容量超限 |
| ContentLengthMismatchException | 400 | Content-Length 与实际大小不一致 |

### 响应格式

统一返回 `ErrorMessage` 对象（JSON）：
```json
{
  "code": "ERROR_CODE",
  "message": "错误描述"
}
```

### 日志策略

- 系统异常（DmsSystemException）：始终打印日志
- 业务异常（DmsBusinessException）：通过 `ExceptionLogProperties.printBusinessException` 控制是否打印
