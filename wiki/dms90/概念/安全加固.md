# 安全加固

> DMS 的安全加固包括路径参数校验（SecureUtil）、XML 防注入（XmlUtil）、文件上传校验（FileUtil）、CSRF 防护、多租户鉴权、权限控制等多层防护。

## SecureUtil — 路径参数校验（dms-common/util 包）

### checkParams() 方法

校验所有路径参数（URI、标签、模板），防止目录遍历攻击。

### 校验规则

| 参数 | 规则 |
|------|------|
| label | 非空、非 null、不含特殊字符 |
| resourceType | 非空、非 null |
| parentFolderId | 存在性检查（调用 folderDao.retrieve()） |
| template | 存在性检查（调用 resourceDao.retrieve()） |

### 防御模式

- 拒绝 `../`、`..\\` 目录遍历
- 拒绝绝对路径
- 拒绝 null/空字符串

## XmlUtil — XML 防注入（dms-common/util 包）

### unmarshal() 方法

解析 XML 时启用 SAX 防护：

```java
spf.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);
spf.setFeature("http://xml.org/sax/features/external-general-entities", false);
spf.setFeature("http://xml.org/sax/features/external-parameter-entities", false);
```

### 防御目标

- XXE（XML External Entity）注入
- XXE 攻击导致文件读取、SSRF、DoS

## FileUtil — 文件上传校验（dms-common/util 包）

### 校验项

| 校验项 | 说明 |
|--------|------|
| 文件名 | 拒绝含路径的文件名（如 `../../etc/passwd`） |
| 文件大小 | 结合 CapacityManager 预检 |
| 文件类型 | 通过后缀名白名单校验 |

### 防御模式

- 拒绝绝对路径
- 拒绝路径遍历字符
- 限制上传文件大小

## CSRF 防护（dms-web/filter 包）

### DMSCsrfFilter

- **生效条件**：`ftf.dms.mode=integrated`
- **CSRF Token**：加载或生成 Token，设置到响应 Header
- **校验规则**：
  - 无 session 的请求直接放行
  - GET/HEAD/TRACE/OPTIONS 方法不检查
  - API 白名单不检查（`/crt`、`/release`、`/delegate` 等）
  - 其他请求校验 Header 或参数中的 Token

### 白名单路径

`/crt`、`/release`、`/prerelease`、`/publish`、`/delegate`、`/doc`、`/file`、`/folder`、`/history-template`、`/material`、`/search`、`/resources`、`/label`、`/metadata`、`/dms/services`、`/dms/rest_v2`、`/dms/j_spring_security_check`

## 多租户鉴权（TenantAuthUtil）

### auth() 方法

校验当前用户是否有权访问目标用户的数据：

```
当前用户 userCode == 目标用户 userCode → 允许
当前用户角色 == ADMIN → 允许
其他 → 抛出 UnAuthorizedException
```

### 使用场景

- FolderService 所有操作
- ResourceService 所有操作
- DocumentService/FileService 部分操作

## 权限控制（RoleController / TenantController / UserController）

### 手动角色校验

```java
User user = UserContext.getCurrent().getUser();
if (!"ADMIN".equals(user.getRole())) {
    throw new UnAuthorizedException("仅 ADMIN 可操作");
}
```

### 受保护的操作

- TenantController：CRUD 操作
- UserController：CRUD 操作（除 `/my` 和 `/tenant`）
- RoleController：角色列表查询

## AdminCheckFilter

- **生效条件**：`ftf.dms.mongo.multitenant-enabled=true`
- **规则**：从 HTTP Basic 认证头中提取用户名，若为 `admin`（不区分大小写），返回 403 FORBIDDEN
- **目的**：禁止使用 admin 账号进行多租户操作

## 文件类型白名单（FileTypeInterceptor）

### 校验逻辑

- 检查 multipart 请求中每个文件的后缀名
- 不在白名单中的文件类型抛出 IllegalInputException

### 配置

`ftf.dms.web.upload-limit.file-type`：允许的文件后缀列表
