# 多租户隔离

> DMS 支持两种租户隔离模式，通过 `ftf.dms.mongo.multitenant-enabled` 配置切换。

## 两种模式对比

| 维度 | 共享模式（默认） | 独立模式 |
|------|-----------------|---------|
| 配置 | `multiTenantEnabled=false` | `multiTenantEnabled=true` |
| 数据库 | 公共 MongoDB | 每租户独立 DB `data_{userCode}` |
| 集合 | `Document-{shardKey}` | `data_{userCode}.Document-{shardKey}` |
| ID 格式 | `{shardKey}-{docId}` | `{userCode}-{shardKey}-{docId}` |
| 查询方式 | `documentService.retrieve(id)` | `documentService.retrieveMultiTenant(id)` |

> **注意**：userCode 是业务用户标识，userId 是系统内部标识。两者通常相同，数据库名称为 `data_{userCode}`（用户级）或 `tenant_{tenantId}`（租户级）。

## ID 解析差异

```
共享模式 ID: 20263-507f1f77bcf86cd799439011
  → shardKey=20263, docId=507f1f77bcf86cd799439011

独立模式 ID: user123-20263-507f1f77bcf86cd799439011
  → userCode=user123, shardKey=20263, docId=507f1f77bcf86cd799439011
  → 路由到 DB: data_user123
```

## 鉴权机制

- 父目录鉴权：`folderService.retrieve(parentFolderId)` → `TenantAuthUtil.auth()`
- 模板鉴权：`resourceService.retrieve(template)` → `TenantAuthUtil.auth()`

## Oracle 层租户模型

DMS 在 Oracle 中通过 `DMS_TENANT` 和 `DMS_USER` 两张表实现租户管理：

- **DMS_TENANT**：存储租户 ID、存储配额（TOTAL_SIZE / USED_SIZE）
- **DMS_USER**：用户名为主键，`TENANT` 字段外键引用 `DMS_TENANT.ID`
- 一个租户下包含多个用户，共享存储配额

> 详见 [[wiki/dms90/来源/2026-04-21 DMS关键表结构|DMS 关键表结构]]。

## 中国电信 DFS 迁移

特殊模式：`ftf.dms.china.telecom.mode=true` + 超过 `dfsTime` 后，从 DFS 路径读取存量文件，实现平滑迁移。
