# Obsidian 图谱视图配置说明

> DMS90 知识库的图谱视图优化配置（v2 — 激进优化版）

---

## ⚠️ 重要：修改配置前必须先关闭 Obsidian

Obsidian 运行时会覆盖 `graph.json`，所以：

1. **先完全退出 Obsidian**（不是关闭窗口，是退出进程）
2. 再修改 `.obsidian/graph.json`
3. 重新启动 Obsidian

---

## 📊 当前配置（v2 激进优化）

### 核心参数

| 参数 | 默认值 | 当前值 | 说明 |
|------|--------|--------|------|
| **max-depth** | 无限制 | **1** | 只显示直接关联 |
| **max-nodes** | 无限制 | **50** | 最多 50 个节点 |
| **nodeSizeMultiplier** | 1 | **0.7** | 节点缩小 30% |
| **lineSizeMultiplier** | 1 | **0.7** | 连线缩小 30% |
| **hideUnresolved** | false | **true** | 隐藏未解析链接 |
| **showOrphans** | true | **false** | 隐藏孤立页面 |
| **collapse-display** | false | **true** | 折叠显示设置 |
| **collapse-forces** | false | **true** | 折叠力导向设置 |
| **linkStrength** | 1 | **0.5** | 降低连线强度（减少抖动） |
| **repelStrength** | 8 | **10** | 增加排斥力（节点更分散） |
| **centerStrength** | 0.3 | **0.5** | 增加向心力（节点更集中） |

### 过滤器

只显示 `wiki/` 目录下的文件，排除 `index.md`、`log.md`：

```
path:wiki -path:[[wiki/index]] -path:[[wiki/log]]
```

### 分组（2 个精简分组）

| 分组 | 颜色 | 包含内容 |
|------|------|---------|
| **概念** 🔵 | 蓝色 | 概念页（shardKey、多租户、限流等） |
| **实体** 🟠 | 橙色 | 实体页（MongoDB、JasperReports 等） |

> v1 有 5 个分组，v2 精简为 2 个核心分组，减少计算量

## 🔧 如果还是卡：手动操作步骤

### 步骤 1：在 Obsidian 中排除 raw 目录

1. 打开 Obsidian → 设置 → 文件与链接
2. 找到"排除的文件"（Excluded files）
3. 添加：`raw`
4. 重启 Obsidian

**效果**：Obsidian 不再索引 `raw/` 目录（63 个文件），搜索和图谱都会更快

### 步骤 2：在图谱视图中手动调整

1. 打开图谱视图
2. 点击右上角 ⚙️ 设置图标
3. 调整：
   - **Depth**: 拖到 `1`
   - **Max nodes**: 拖到 `50` 或更低
   - **Neighbors**: 拖到最低
4. 右侧分组面板：只勾选"概念"和"实体"

### 步骤 3：使用迷你图谱代替全局图谱

- 打开任意页面 → 右上角迷你图谱
- 只显示当前页面的直接关联
- **最轻量，绝不卡**

## 🎯 推荐工作流

```
日常使用（绝不卡）
├─ 查看某个概念的关联 → 使用"迷你图谱"
├─ 查看整体架构 → 图谱 → 只勾选"概念"+"实体"
└─ 排查问题 → Ctrl+Shift+F 搜索（比图谱更快）

偶尔使用
└─ 需要全局视图 → 打开图谱 → 勾选更多分组
```

## ⚡ 应急方案

如果 Obsidian 完全卡死：

1. **任务管理器** → 结束 Obsidian 进程
2. 删除 `.obsidian/workspace.json`（重置工作区）
3. 重新启动 Obsidian
4. **不要直接打开图谱**，先打开一个普通页面
5. 确认正常后再打开图谱
