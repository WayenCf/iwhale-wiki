# 批量账单生成流程

> DMS 的批量账单生成是核心业务流程，从定时扫描 XML 文件开始，经过解析、任务拆分、并发生成、上传、建索引、打标签，最终生成 PDF 账单。

## 完整流程图

```
定时扫描器(ScheduledXmlDataScanner)
    → 发现新 XML 文件
    → TaskMonitor.mark() 记录为 RUNNING
    → BatchDocumentServiceImpl.make(xmlPath)
        → move() 移动到 workspace 目录
        → DocMakeTaskFactory 解析 XML
            → 按 splitElement 拆分为多个 Node
            → 每个 Node 提取 docId / data / template / format / locale / shardKey / crt / labels
            → 返回 DocMakeTask 迭代器
        → 逐个提交 docMakingExecutor.execute(docMakeTask)
        → DocMakeTask.call()
            → 获取 Release 中的 Resource Map
            → DocGenerator.generateDoc() 生成 PDF
            → 创建 DocUploadTask 提交 repositoryExecutor
        → DocUploadTask.doUpload()
            → 容量预检 VolumeService.preCheck()
            → PDF 签名（如有证书）PdfUtil.sign()
            → 上传文件到存储
            → 创建 Document 元数据
            → 建索引 IndexService.create()
            → 创建标签 documentLabelService.create()
        → TaskRecordServiceImpl 更新成功/失败计数
        → 扫描器检查 SUCCEEDED 后删除源 XML
```

## 核心组件

### 1. 定时扫描器（ScheduledXmlDataScanner）

- **触发条件**：`ftf.dms.batch.xmlScanner.enabled=true`
- **扫描路径**：`{xmlDir}/{tenant}/{userId}/`
- **过滤规则**：
  - 按 `unmatch` 正则跳过不匹配的文件
  - `limitXmlSize` 启用时检查文件大小不超过 `maxMemory / 20`
- **状态管理**：`taskMonitor.mark()` 标记为 RUNNING

### 2. BatchDocumentServiceImpl

- **方法**：`make(url)`
- **步骤**：
  1. 将 XML 文件从原目录 move 到 workspace 目录（防止重复处理）
  2. 创建 DocMakeTaskFactory 解析 XML
  3. 逐个提交到 `docMakingExecutor` 线程池执行
  4. 每个文档生成后创建标签
  5. 记录总数 `taskRecordService.total(url, count)`

### 3. DocMakeTaskFactory

- **职责**：XML 解析核心工厂
- **步骤**：
  1. SAX 解析 XML 文件（启用 `disallow-doctype-decl` 防 XXE）
  2. 提取全局参数节点（GlobalTemplateElement 等）
  3. 按 `splitElement` XPath 选出所有分割节点
  4. 每个 `take()` 调用组装一个 DocMakeTask

### 4. DocMakeTask

- **职责**：单个文档生成任务
- **步骤**：
  1. 获取 Release 中的 Resource Map
  2. 调用 DocGenerator.generateDoc() 生成 PDF
  3. 创建 DocUploadTask 提交 repositoryExecutor

### 5. DocUploadTask

- **职责**：文档上传任务
- **步骤**：
  1. 容量预检 VolumeService.preCheck()
  2. PDF 签名（如有证书）
  3. 上传文件到存储（MongoDB GridFS 或 Spool）
  4. 创建 Document 元数据
  5. 建索引 IndexService.create()
  6. 创建标签 documentLabelService.create()

### 6. TaskRecordServiceImpl

- **职责**：任务执行记录
- **内部类 TaskStatistics**：
  - AtomicInteger 线程安全计数（total/success/fail）
  - Vector<FailResultWrapper> 线程安全失败列表
- **持久化策略**：每完成 20 个任务或全部完成时写入数据库

## 并发控制

### 线程池

| 线程池 | 配置前缀 | 用途 |
|--------|---------|------|
| docMakingExecutor | `ftf.dms.executor.docmaking.*` | 文档生成 |
| repositoryExecutor | `ftf.dms.executor.repository.*` | 文档落盘 |
| batchDocMakingExecutor | - | 新接口文档生成 |
| sendMqMessageExecutor | - | MQ 消息发送 |

### DmsExecutor

- **类型**：自定义 ThreadPoolExecutor
- **队列**：LinkedBlockingDeque
- **策略**：CallerRunsPolicy（满时由调用线程执行）
- **ThreadLocal**：ThreadLocalUserHolder 保持用户上下文传递

## 错误处理

### TaskMonitor

- `mark()`：首次创建 TaskRecord
- `scheduleFailed()`：调度级别失败，直接设为 FAILED
- 失败/部分失败时允许重新处理

### BaseEntityTask

- `onFailure()`：任务级别失败，更新 TaskRecord 或发送 MQ 消息

### 文件移动

- `FileNotFoundException` 静默处理（多实例竞争场景）

## 大账单切割（HeavyDocMakeTask）

### 切割流程

1. 解析切割规则（JSON → Rule → SegmentDef）
2. 创建 DivisibleXml 迭代器
3. 逐段切割 → 每段生成 PDF → 保存到临时目录
4. 用 PDFMergerUtility 合并所有分段 PDF
5. 创建 HeavyDocUploadTask 上传合并后的 PDF
6. 清理临时目录

### 切割规则

- **GlobalSegmentDef**：全局段（不随分割节点重复）
- **HeaderSegmentDef**：头部段（分割节点之前的区域）
- **DetailSegmentDef**：明细段（每个分割节点对应一段，支持 groupSize 分组）
- **TailSegmentDef**：尾部段（分割节点之后的区域）

## 配置属性（BatchProperties）

| 属性 | 说明 |
|------|------|
| xmlDir | XML 目录 |
| splitElement | 分割节点 |
| docIdElements | 构成 docId 的节点列表 |
| dataElement | 数据节点 |
| fileNameElement | 文件名节点 |
| templateElement / globalTemplateElement | 模板节点 |
| defaultTemplate | 默认模板 |
| shardKeyElement / globalShardKeyElement | 分表键节点 |
| shardingEnabled | 是否启用分表 |
| initialDelay / period / timeUnit | 调度参数 |
| limitXmlSize | 限制 XML 大小 |
| forceOverrideExistedDoc | 强制覆盖已存在文档 |
