# 实体继承体系

> DMS 的实体采用三层继承体系，从基类 BaseEntity 到 UserBaseEntity 再到具体业务实体 File/Document，贯穿租户隔离、用户上下文、分表路由等核心概念。

## 继承层次

```
BaseEntity（基类）
  └── UserBaseEntity（租户+用户基类）
        ├── File（文件）
        │     └── Document（文档）
        └── 其他业务实体...
```

## BaseEntity（基类）

### 核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | String | MongoDB ObjectId 或复合 ID |
| creationDate | Date | 创建时间 |
| updateDate | Date | 更新时间 |

### 核心方法

| 方法 | 说明 |
|------|------|
| `getShardKey()` | 返回分表键，由子类实现 |
| `get_id()` | 返回复合 ID `{shardKey}-{realId}` |
| `isLarge()` | 判断是否大文件 |

## UserBaseEntity（租户+用户基类）

### 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| user | String | 用户编码（userCode） |
| tenant | String | 租户编码 |
| labels | Set<Label> | 标签集合 |
| shardKey | String | 分表键 |

### 核心方法

| 方法 | 说明 |
|------|------|
| `getShardKey()` | 返回 shardKey 字段 |
| `getTenant()` | 返回租户编码 |

## File（文件基类）

### 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| fileId | String | GridFS 文件 ID（ObjectId） |
| fileName | String | 文件名 |
| contentType | String | MIME 类型 |
| length | Long | 文件大小（字节） |
| parentFolderId | String | 父文件夹 ID |
| storePath | String | 存储路径（Spool 模式） |

### 关联关系

- 一对一关联到 `FileAttach`（GridFS 存储）
- 一对多关联到 `FileLabel`（标签）

## Document（文档类）

### 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| template | String | 模板 URI |
| locale | String | 区域设置（如 en、zh） |
| format | String | 输出格式（pdf/html/csv） |
| status | String | 文档状态 |

### 关联关系

- 一对一关联到 `DocumentAttach`（GridFS 存储）
- 一对多关联到 `DocumentLabel`（标签）
- 多对一关联到 `Resource`（模板）

## 其他业务实体

| 实体 | 继承 | 用途 |
|------|------|------|
| Folder | UserBaseEntity | 文件夹 |
| Resource | UserBaseEntity | 模板资源 |
| Release | UserBaseEntity | 发布版本 |
| CertificateAndKey | UserBaseEntity | 数字证书 |
| AccessLog | UserBaseEntity + TenantAware | 访问日志 |
| TaskRecord | UserBaseEntity | 任务记录 |

## 与 shardKey 分表的集成

每个实体通过 `getShardKey()` 方法决定存储到哪个集合：

```
写入时：entity.getShardKey() → 选择集合 Document-{shardKey}
读取时：IdUtil.getKeyAndId(id) → 提取 shardKey → 选择集合
```

## 与多租户的集成

```
非多租户：ID 格式 {shardKey}-{realId}
多租户：ID 格式 {userCode}-{shardKey}-{realId}
```

UserBaseEntity 的 user 字段在多租户模式下作为 ID 第一段。
