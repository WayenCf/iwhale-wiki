# 双存储适配

> DMS 支持双存储后端（MongoDB/Spool 云盘）同时部署，通过 DaoFactory 实现运行时存储路由，支持 MongoDB 优先、Spool 降级、混合部署三种模式。

## 存储类型枚举（StorageType）

| 值 | 说明 |
|----|------|
| SPOOL | Oracle/SPOOL 存储（云盘） |
| MONGODB | MongoDB 存储 |

## DaoFactory 路由逻辑

```java
static <T> T getDao(Integer storageType, Map<StorageType, T> daoMap)
```

### 路由规则

| 模式 | storageType | 目标 DAO |
|------|-------------|---------|
| 混合模式 | null 或 2 | MONGODB（优先） |
| 混合模式 | 其他 | SPOOL（降级） |
| 纯 SPOOL | 任意 | SPOOL |
| 纯 MONGO | 任意 | MONGODB |
| 默认 | 任意 | MONGODB |

### 配置控制

- `DEPEND_SPOOL_AND_MONGO=true`：启用混合模式
- `@Profile("mongodb")` / `@Profile("spool")`：Profile 激活单模式

## DocumentServiceImpl/FileServiceImpl 双存储逻辑

### CUD 操作（创建/更新/删除）

```
尝试 MongoDB 操作
  → 成功 → 返回
  → 失败 → fallback 到 Spool 云盘
```

### R 操作（查询）

```
混合模式（DEPEND_SPOOL_AND_MONGO）：
  1. 先查 MongoDB
  2. 查不到 → 查 Spool 云盘
  3. 合并结果

单模式：直接查询对应存储
```

### 典型代码模式

```java
// 双存储查询
Document doc = documentDao.retrieve(id);
if (doc == null && DEPEND_SPOOL_AND_MONGO) {
    doc = spoolDocumentDao.retrieve(id);
}
```

## Spool 云盘集成

### 存储路径模式

- `storePath` 字段记录云盘相对路径
- 上传：`FileUtil.copyInputStreamToFile(source, new File(storePath))`
- 下载：`new FileInputStream(new File(storePath))`
- 删除：`File file = new File(storePath); file.delete()`

### 容量管理

- `VolumeService.preCheck()`：上传前 CAS 预留容量
- `VolumeService.retrieveTenantVolume()`：通过 Redis Hash 统计已用容量

### DFS 迁移模式

- `ftf.dms.china.telecom.mode=true` + 超过 `dfsTime` 后
- `renameDfsDoc()`：从 DFS 历史路径读取存量文件
- 实现平滑迁移：新文件 MongoDB，存量文件 DFS

## 混合部署场景

### 典型配置

```properties
# 启用混合模式
dms.depend.spool.and.mongo=true

# MongoDB 连接
spring.data.mongodb.uri=mongodb://host:port/database

# Spool 云盘路径
dms.spool.base.path=/mnt/spool
```

### 优势

- **高可用**：MongoDB 挂了自动降级到云盘
- **平滑迁移**：从云盘逐步迁移到 MongoDB
- **性能分层**：热数据 MongoDB，冷数据云盘
