# 限流机制（Sentinel）

> DMS 使用 Alibaba Sentinel 1.7.2 实现上传/下载的并发控制，防止单一客户端占用过多资源。

## 限流范围

| 操作 | 限流方法 | 超限行为 |
|------|---------|---------|
| 文档生成/上传 | `executeUpload()` | HTTP 429 `UploadTooManyRequestsException` |
| 文件上传（按大小） | `executeNonContentUpload(fileSize)` | 同上 |
| 文档/文件下载 | `executeDownload()` | 同上 |
| token=true 获取 URL | **不限流** | — |

## 配置参数

**配置文件位置**：`dms-web/ZSMART_HOME/etc/dmsConfig.properties`

| 参数 | 说明 |
|------|------|
| `ftf.dms.upload.concurrent.threads` | 上传并发线程上限 |
| `ftf.dms.download.concurrent.threads` | 下载并发线程上限 |
| `ftf.dms.upload.total.max-bytes` | 最大并发上传字节数 |

## 额外控制

- **Content-Length 欺骗检测**：`uploadConfig.strictContentLengthCheck` 开启后，验证声明大小与实际大小的一致性
- **容量预检**：`volumeService.preCheck()` 写入前校验租户剩余容量
- **容量回退**：写入失败调用 `volumeService.rollback()` 回退计数

## 限流在流程中的位置

```
上传请求 → Sentinel 限流 → 安全校验 → 容量预检 → 业务处理
下载请求 → Sentinel 限流（token=false时） → 文件传输
```
