# 模版导入

> DMS 模版从设计器导出后导入到目标环境的流程，涉及数据库写入和资源关联。

## 在现场问题中的角色

### 用户不一致

- [[wiki/dms90/来源/2024-03-21 GAIA项目账单生成异常]]：测试环境用 A 用户设计模版，导入其他环境后没有 A 用户，导致无法生成账单
- 解决：设计模版使用特殊账号保证各环境都存在

### 数据库连接中断

- [[wiki/dms90/来源/2024-10-12 PTO配置环境导入模板报错]]：TeleDB 节点故障导致导入报错
- [[wiki/dms90/来源/2025-03-10 模版导入问题汇总]]：同样 TeleDB 连接问题

### CSRF 拦截

- [[wiki/dms90/来源/2025-03-10 模版导入问题汇总]]：未配置 `csrf-url-skip` 导致 CSRF 校验拦截

### CLOB 字段超长

- [[wiki/dms90/来源/2025-03-10 模版导入问题汇总]]：`DMS_RESOURCE.DEPENDENCIES` 字段超长，ORA-22275

### DMS_RESOURCE 表结构要点

- **ID 即模版路径**：模版在前台的访问路径就是数据库主键
- **DEPENDENCIES**：CLOB 类型，存储依赖项 ID，超长触发 ORA-22275
- **FILE_ID**：外键引用 `dms_resource_attach`，关联模版文件
- **PERMISSION_MASK**：权限掩码，控制模版访问权限
- 详见 [[wiki/dms90/来源/2026-04-21 DMS关键表结构|DMS 关键表结构]]

### 模版内容格式问题

- [[wiki/dms90/来源/2024-04-08 尼泊尔Ncell发票内容格式不对]]：导入后预览与设计器不一致，字段显示格式问题

## 常见故障模式

1. **用户不存在** → 模版无法使用 → 使用通用账号设计
2. **数据库连接断** → 导入失败 → 切换数据库节点
3. **CSRF 拦截** → 导入请求被拒 → 配置 `csrf-url-skip`
4. **CLOB 超长** → SQL 执行失败 → 手动数据库更新
