# 离线账单生成

> DMS 批量处理 XML 文件生成账单 PDF 的功能，通过扫描共享目录实现。

## 在现场问题中的角色

### task_record 卡在 running

- [[wiki/dms90/来源/2024-03-12 卢旺达文件上传下载失败]]：CUST_BILL_ADDR 字段超长导致 JasperReports 死循环
- [[wiki/dms90/来源/2024-08-23 MPT批量XML上传接口Running]]：同样问题
- [[wiki/dms90/来源/2024-05-14 尼泊尔Ncell批量处理XML部分失败]]：XML 特殊字符导致部分解析失败
- [[wiki/dms90/来源/2024-06-04 XML特殊字符导致PDF生成失败]]：XML 中含特殊字符未转义导致解析失败

### 配置错误

- [[wiki/dms90/来源/2024-04-02 DMS无法生成PDF到指定目录]]：配置项名称错误，使用了不存在的配置名

### 本地验证

- [[wiki/dms90/来源/2026-03-12 阿富汗ETA离线生成账单失败]]：通过本地模拟验证确认功能正常

## 处理流程

```
业务生成XML → 上传共享目录 → DMS扫描 → 移动到workspace → 记录running → 生成PDF → 记录success
```

## 关键配置

| 配置项 | 说明 |
|--------|------|
| `ftf.dms.batch.xml-dir` | XML 扫描目录 |
| `ftf.dms.batch.globalDynamicFolderDocSavePath` | 动态保存目录 |
| `ftf.dms.batch.executor-pool-size` | 并发处理线程数 |
| `ftf.dms.batch.executor-time-out` | 处理超时时间 |

## 常见故障模式

1. **字段超长** → JasperReports 死循环 → task_record 卡在 running
2. **配置错误** → PDF 无法生成到指定目录
3. **XML 特殊字符** → 解析失败

## 故障排查步骤

1. 检查 `dms_task_record` 表是否有 running 状态
2. 删除 running 记录，重启服务
3. 获取现场模版本地验证
4. 检查 XML 是否含特殊字符
