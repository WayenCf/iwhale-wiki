# 现场问题快速检索指南

> 遇到现场问题时，按本指南快速定位已有解决方案，避免重复踩坑。

---

## 方法一：关键词搜索（最快）

Obsidian 内置搜索 `Ctrl+Shift+F`，支持多种语法：

| 场景 | 搜索语法 | 示例 |
|------|---------|------|
| 按错误信息搜 | `"错误关键字"` | `"Timed out after 30000 ms"` |
| 按项目名搜 | `tag:#现场问题 项目名` | `tag:#现场问题 阿富汗` |
| 按组件搜 | `tag:#现场问题 组件名` | `tag:#现场问题 MongoDB` |
| 按问题类型搜 | `path:wiki/dms90/概念 关键词` | `path:wiki/dms90/概念 超时` |

## 方法四：Dataview 查询（结构化筛选）

安装 Dataview 插件后，在任意笔记中插入查询块：

### 按项目筛选

```dataview
TABLE WITHOUT ID
  file.link AS "问题",
  choice(contains(file.tags, "现场问题"), "🔴", "⚪") AS "类型"
FROM "wiki/dms90/来源"
WHERE contains(file.name, "阿富汗")
SORT file.name DESC
```

### 按组件筛选

```dataview
TABLE WITHOUT ID
  file.link AS "问题"
FROM "wiki/dms90/来源"
WHERE contains(file.tags, "MongoDB")
SORT file.name DESC
```

## 推荐工作流

遇到新现场问题时，按以下流程检索：

```
1. 搜索 → Ctrl+Shift+F 输入错误关键字
   ↓ 找到类似问题？
   ├── 是 → 查看解决方案，直接应用
   └── 否 → 进入下一步

2. 定位 → 查上面的"高频概念页速查表"
   ↓ 找到对应概念页？
   ├── 是 → 查看关联的现场问题，看是否有类似解法
   └── 否 → 进入下一步

3. 问我 → 把问题描述发给 Claude
   → Claude 会检索整个知识库并综合回答
   → 解决后，把新问题摄入知识库（避免下次再踩坑）
```
